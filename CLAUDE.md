# CLAUDE.md - 기업 홈페이지 프로젝트 가이드라인

## 기술 스택

- **빌드 도구:** Vite
- **프레임워크:** React (최신 안정 버전 - 중요: 알려진 보안 취약점(CVE)이 없는 버전 필수 사용)
- **언어:** TypeScript
- **린팅/포매팅:** ESLint, Prettier

## 주요 명령어

- **개발 서버:** `npm run dev`
- **빌드:** `npm run build`
- **프리뷰:** `npm run preview`
- **린트:** `npm run lint`
- **포맷:** `npm run format` (또는 `npx prettier --write .`)
- **보안 점검:** `npm audit`

## 코딩 표준

- **함수형 컴포넌트:** React 함수형 컴포넌트와 Hook을 사용합니다. 클래스 컴포넌트는 지양합니다.
- **TypeScript:** strict 모드를 활성화합니다. `any` 타입 사용을 지양하고, 모든 Props와 상태(State)에 대해 interface/type을 정의합니다.
- **스타일링:** CSS Modules 또는 CSS-in-JS를 사용합니다. 일관된 명명 규칙을 유지합니다.
- **반응형 디자인:** 다양한 화면 크기(데스크탑, 태블릿, 모바일)에 대응하는 유연한 UI를 구현하며, 특히 모바일 환경에서도 불편함이 없도록 가독성과 조작성을 고려합니다.
- **상태 관리:** 간단한 요구사항은 local state/Context API를 사용하며, 과도한 엔지니어링을 피합니다.

## 보안 가이드라인 (중요)

1. **React 버전:** 최근 보안 이슈(XSS, prototype pollution 등)가 해결된 안전한 버전을 사용해야 합니다. 설치 후 반드시 `npm audit`으로 확인하십시오.
2. **의존성 관리:** 정기적으로 `npm audit`을 실행하여 취약점이 발견된 패키지를 업데이트합니다.
3. **입력값 처리:** 모든 사용자 입력값은 반드시 검증 및 살균(Sanitize) 처리를 거칩니다.
4. **위험 요소:** `dangerouslySetInnerHTML` 사용은 가급적 피하며, 필요시 반드시 철저한 살균 과정을 거쳐야 합니다.
5. **비밀 정보:** API 키나 민감한 정보를 저장소에 커밋하지 마십시오. `.env` 파일을 활용합니다.

## 프로젝트 컨텍스트

- **유형:** 기업 홈페이지 (Corporate Homepage)
- **목표:** 전문적이고 안전하며 성능이 뛰어난 기업 브랜드 구현.
- **UI/UX:** 직관적이고 사용자 친화적인 경험을 제공하며, 특히 모바일 환경에서의 가독성과 조작 편의성을 최우선으로 고려합니다.
- **구조:** 표준 Vite + React 폴더 구조를 따릅니다 (`src/components`, `src/assets`, `src/pages` 등).
